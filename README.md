##  製品概要

本製品は、**ハイブリッド暗号方式（RSAおよびAES）** を採用した、スタンドアロン型のパスワード管理アプリケーションです。
JavaFXによる直感的なGUIを提供し、外部のクラウドサーバー等に依存することなく、すべての認証データおよび暗号化キーをユーザーのローカル環境でのみ安全に一元管理します。

###  主な特徴

- **強固な暗号化アルゴリズム**
  - アカウント情報（ユーザー名・パスワード）の暗号化： `RSA (2048bit)`
  - RSA秘密鍵の保護： `AES-GCM (128bit)` + `IV (96bit)`
  - マスターパスワードの鍵導出： `PBKDF2WithHmacSHA256` (反復回数: 65,536回, ソルト付き)
- **完全なゼロ知識・ローカル管理**
  - ネットワーク通信を一切行わず、認証情報はすべてローカルにJSON形式で暗号化保存されます。
  - アプリ自身もマスターパスワードを保持しないため、開発者であってもデータの復元は不可能です。
- **データのポータビリティ**
  - 実行場所に関わらず、常に一定の絶対パス（`${user.home}/.rsa_password_tool/`）を参照します。
  - これにより、USBメモリ等からアプリを起動しても、PC内の同一データソースに安全にアクセスできます。

##  データ保存先とファイル構成

初回セットアップ時、ユーザーのホームディレクトリ直下に専用の隠しフォルダが自動生成され、以下のファイルが配置されます。

**保存ディレクトリ:**
- Windows: `C:\Users\[UserName]\.rsa_password_tool\`
- Mac/Linux: `~/.rsa_password_tool/`

| ファイル名 | フォーマット | 役割 |
| :--- | :--- | :--- |
| `passwords.json` | JSON | 暗号化されたログイン情報（タイトル、ユーザー名、パスワード）のリスト。 |
| `private_key.bin` | バイナリ | マスターパスワード由来のAES鍵で暗号化されたRSA秘密鍵。 |
| `public_key.txt` | テキスト | 暗号化に使用するRSA公開鍵（平文保存）。 |
| `salt.bin` | バイナリ | PBKDF2の鍵導出で使用する16バイトのソルト値。 |
| `recovery_key.bin` | バイナリ | 初回発行される「復旧キー（UUID）」で暗号化されたRSA秘密鍵。 |

## インストールと実行方法

### 一般ユーザー向け（ビルド済みJARの利用）
Releases ページから `rsa-password-1.0-SNAPSHOT-jar-with-dependencies.jar` をダウンロードし、以下のいずれかで実行してください。

- **GUIで起動**: `.jar` ファイルをダブルクリック
- **CLIで起動**:
  ```bash
  java -jar rsa-password-1.0-SNAPSHOT-jar-with-dependencies.jar

### 開発者向け（ソースコードからのビルド）
1 リポジトリをクローンします。
    Bash
    git clone [https://github.com/あなたのユーザー名/rsa-password-manager.git](https://github.com/あなたのユーザー名/rsa-password-manager.git)
    cd rsa-password-manager
2 開発環境での直接実行（JavaFX起動）
    Bash
    mvn clean compile javafx:run
3 配布用JARファイルの生成
    Bash
    mvn clean package
    ※ビルド成功後、target/ ディレクトリ内に依存関係を含む実行可能JARが生成されます
## 主要機能
1 初回セットアップ
    マスターパスワードの設定と、RSA鍵ペア、AES鍵、ソルトの自動生成。
    万が一のパスワード忘れに備えた「復旧キー（リカバリーキー）」の発行。
2 ダッシュボード・自動復号表示
    登録済みのアカウント一覧を表示。
    リストから選択すると、メモリ上で安全にRSA復号が行われ、ユーザー名とパスワードをクリアテキストで表示。
3 新規登録・削除
    新しいアカウント情報をRSA公開鍵で個別に暗号化して保存。
    不要なデータの物理削除。
4 リカバリー（復旧）機能
    マスターパスワードを忘れた場合、初回セットアップ時に発行された「復旧キー」を用いて秘密鍵を復号し、新しいマスターパスワードの再設定が可能。
5 システム初期化（ファクトリーリセット）
    マスターパスワード認証後、絶対パスで管理されているすべての鍵およびデータファイルを物理削除し、初期状態に戻します。

## セキュリティに関する重要なお願い
1 マスターパスワードの厳重保管
    マスターパスワードはシステムのどこにも保存されません。忘却し、かつ「復旧キー」も紛失した場合、データは永遠に復号できなくなります（救済措置はありません）。
2 復旧キーのオフライン管理
    初回セットアップ時に画面に一度だけ表示される「復旧キー」は、デジタル媒体ではなく紙のメモなどに記録し、安全な場所に保管することを強く推奨します。
3 リポジトリへのコミット厳禁
    開発を行う際、~/.rsa_password_tool/ 内の鍵ファイルや passwords.json を誤ってGitHubにコミットしないでください（本リポジトリの .gitignore に除外設定が含まれています）。
